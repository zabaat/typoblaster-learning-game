# Game Designer Perspective

## Role Summary
Protect the core loop. Typing, projectile, hit, score, threat, replay. Add depth only after that loop feels good.

## Build Guidance
- Preserve the MVP.
- Make the core interaction satisfying.
- Avoid unnecessary systems.
- Ensure this perspective is represented in implementation decisions.

## Acceptance Lens
Before calling the MVP complete, ask:
1. Does the game communicate its premise quickly?
2. Does the player feel immediate feedback?
3. Does the system support endless randomized play?
4. Is this still simple enough to maintain?
